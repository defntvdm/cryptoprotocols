Реализация протокола подписи чека без разглашения id

Протокол позволяет на основе большого количества чеков с одинаковым содержимым,
но разными id, поднять степень доверия банка, за счёт чего банк подпишет наш
чек неглядя.

1) Пользователь генерирует N чеков с различными ID и одинаковым содержимым
2) Пользователь отдаёт чеки банку
3) Банк проверяет N-1 чек. С проверкой каждого чека степень доверия
  банка растёт.
4) Банк подписывает последний чек неглядя
5) Пользователь довольный, особенно если чек был с увеличенной суммой перевода

Ключи: 
    -n - количество чеков
    -с - скомпрометировать 1 чек
